Release Notes

1.0.14.001

This update fixes an issue with playing surround audio files in Windows 7.

  1. In Windows 7, an audio file with more than two channels wouldn't play unless it had a non-zero channel mask with one speaker assigned to each channel. A zero channel mask indicates default speaker assignments and should work, but in Windows 7 it caused a DirectSound error. WaveShop now detects this case and synthesizes a channel mask.
  2. Previous versions could crash while attempting to read a corrupted WAV file, particularly if the file's data chunk was smaller than expected e.g. due to truncation. Fixed.

1.0.13.000

This update gives WaveShop a more modern appearance, and also fixes some minor bugs.

  1. WaveShop now uses Common Controls 6.0, AKA the XP theme. This gives the app a more modern appearance, and makes it more compatible with Windows 7 and Aero.
  2. After opening an AAC or MP4 file, it appeared that the file could be saved in the same format, when in fact WaveShop can only read AAC/MP4 format, not write it. This made it easy to create a file with a file extension of .acc, but actually containing WAV format. Attempting to open such a file in WaveShop gave the error "Channel coupling not yet implemented." Fixed, by forcing a Save As dialog and changing the file extension.
  3. In Windows 7, list controls weren't repainting their columns during column resizing. Dragging a separator in the list header caused a vertical line to be displayed indicating the column width, and column repainting was deferred until the left mouse button was released. In XP this behavior is associated with unchecking the "Show window contents while dragging" setting in Display Properties/Appearance/Effects, but Windows 7 works very differently. Fixed by using Common Controls 6.0; for details, see here.
  4. If the Metadata Editor dialog was resized, the Clear button didn't move. Fixed.

1.0.12.000

WaveShop now supports metadata editing.

1.0.11.002

WaveShop now supports EQ, limiting, compression, and other effects, via LADSPA plugins.

  1. A LADSPA plugin host was added, and the WaveShop download now includes some of Steve Harris' plugins. More plugins are available from the WaveShop download page.
  2. The Spectrum dialog now lets the user optionally specify a frequency range.

1.0.10.000

This update adds channel selection, allowing operations on any subset of the channels in a multi-channel audio file.

  1. Channel selection was added for certain commands.
  2. An "Open recording" checkbox was added to the Record dialog.

1.0.9.004

This update adds one-touch and sound-activated recording, along with frequency sweep generation and clip detection in level meters.

  1. A record options page was added, to allow one-touch recording (start and stop are immediate, with no prompts), sound-activated recording (start and stop are triggered by audio levels meeting specified conditions), and recording via global hot keys.
  2. The /r command line flag was added, which causes WaveShop to execute the record command immediately upon launching. This makes it possible to schedule a recording via the Windows Task Scheduler.
  3. A frequency sweep wizard was added to the audio generator.
  4. The level meters now support clip detection. Left-click on a clip indicator to reset it, or Alt+left-click anywhere within the meters view to reset all clip indicators at once.
  5. Changing the audio format was causing the realtime spectrum analyzer to display incorrect results; fixed.

1.0.8.007

This update adds recording.

  1. WaveShop now supports recording; duration is unlimited (except by disk space).
  2. Changing playback buffer size while playing was failing silently; fixed.
  3. Device selections are now made persistent via GUID instead of description.

1.0.7.002

This update adds AAC decoding via libfaad2.

  1. WaveShop can now open AAC files, and MP4 files containing AAC audio.
  2. The help is now context-sensitive.
  3. The location of temporary files can now be specified; see Options/Edit.

1.0.6.001

This update adds MP3 encoding via LAME. Note that libmp3lame.dll is NOT included in the WaveShop download, so you'll need to obtain it, e.g. from RareWares; directions are here.

1.0.5.000

This update fixes a major bug in the MP3 decoding, as well as some lesser bugs.

  1. The MP3 decoder was introducing periodic artifacts into the audio; fixed.
  2. The spectrum bar could show unexpected output when playback started; fixed.
  3. Zero-length audio could cause an access violation in the meter bar; fixed.

1.0.4.006

This update adds spectral analysis, both static and real-time.

  1. A command was added to analyze the frequency spectrum of audio; see Audio/Spectrum.
  2. A real-time spectrum analyzer was added; see View/More Bars/Spectrum.
  3. In rulers and plots, the number of minor ticks now varies with unit and zoom.
  4. Resizing the RMS histogram caused severe flicker; fixed via double-buffering.
  5. Canceling during RMS statistics reenabled the main menu, allowing reentrance and potentially causing a crash if the document was closed before the RMS dialog; fixed.

1.0.3.003

This update adds peak meters, and channel names in the view.

  1. A peak meter bar was added; it fully supports surround audio.
  2. Channel names can now be optionally displayed in the view.
  3. For 32-bit audio only, RMS Statistics handled the negative rail incorrectly; fixed.
  4. Adjacently docked sizing bars were displaying an unexpected black border; fixed.
  5. Editing speaker assignments caused the audio player to escape its loop; fixed.
  6. Resample, Format, and Delete/Insert Channel were resetting speaker assignments; fixed.

1.0.2.003

This update adds sample rate conversion, MP3 import, and easy file concatenation.

  1. Sample rate conversion is supported via libsamplerate. The command is Audio/Resample.
  2. MP3 import is supported via libmad. The decoder can output 16-bit or 24-bit PCM.
  3. Edit/Insert now inserts one or more audio files. To insert silence, use Edit/Insert Silence.
  4. The Edit/Insert file dialog now allows multiple selection, so it's easy to concatenate many audio files at once. The same applies to concatenating channels via Insert Channel.
  5. The naming convention for mono files generated by Extract Channels has changed: the speaker name is now prefixed by the channel number, so the files sort correctly.
  6. The active document's audio format is now shown in the status bar.
  7. In the Find Clipping command, the clipping level can now be specified, to find peaks.
  8. The VBR encoding quality for writing Ogg/Vorbis can now be changed, in Options/Audio.
  9. Automatic scrolling during selection drag was slow or failed in certain cases; fixed.

1.0.1.003

This update adds features, including support for file formats other than WAV.

  1. WaveShop now reads and writes file formats other than WAV, via libsndfile.
  2. Audio exceeding 4GB is now supported in the 64-bit version.
  3. A history bar was added, allowing undo/redo to a selected state.
  4. The view colors can now be customized, via Options/View.
  5. Insert Channel wasn't showing a progress bar during read; fixed.
  6. Canceling during an undo or redo caused unexpected behavior; fixed.

1.0.0.160

  1. Find Clipping within a selection was misplacing the audio cursor. Fixed.
  2. Resizing the view wasn't adequately constraining the scroll position. Fixed.
  3. Scrolling could cause a double-width or duplicate audio cursor. Fixed.
  4. The Peak Statistics dialog's min/max "Go to" was ignoring selections. Fixed.
  5. The help file is now complete.

1.0.0.159

Minor changes were made to support the portable version.

1.0.0.154

  1. Left-clicking on the time ruler now zooms in, without moving the audio cursor.
  2. F6 now toggles between the view and the navigation bar.
  3. In the Find dialog, the channel combo box wasn't always properly initialized. Fixed.

1.0.0.153

  1. If the Navigation bar's selection start (or end) edit control had input focus, closing the document would crash the app. Fixed.
  2. Moving the cursor to the end of the audio caused it to disappear. Fixed.
  3. Reentrance was potentially occurring in Check for Updates. Fixed.
  4. The Find Clipping results dialog was initially scrolled a few lines. Fixed.
  5. The peak and RMS statistics dialogs now have sortable columns.
  6. The 64-bit version can now be distinguished via Help/About.
  7. A rudimentary help file was added; it's a work in progress.

1.0.0.152

  1. In the Peak Statistics dialog, added a "Goto" context menu for the Min and Max columns.
  2. Added Audio/Invert, to invert selected audio.
  3. Added Edit/Find and Edit/Find Next, to find sample(s) matching specified criteria.
  4. Check for updates wasn't working in Windows 7. Fixed. If you run Window 7, please download and install the latest version, otherwise you won't get any updates.

1.0.0.148

The disk threshold (Options/Edit) default was changed to zero, forcing all clipboard and undo states to disk. This maximizes the memory available for documents and temporary buffers.

1.0.0.147

This beta version adds a few features, and fixes some bugs.

  1. A Change Format command was added to the Audio menu, allowing the channel count, sample rate, and sample size to be changed. This command does NOT resample the audio, so changing the sample rate will change the playback pitch.
  2. Channels can now be inserted into an empty (new) document, via Audio/Insert.
  3. For 8-bit audio only, Edit/Insert was inserting a bogus value instead of silence. Fixed.
  4. Sample sizes that aren't a multiple of 8 bits are better supported now.
  5. The debug console window was accidentally left enabled. Fixed.
  6. Audio sometimes wouldn't play after deleting/inserting channels. Fixed.
